JS 笔记：正则表达式相关

2021-10-10
JavaScript

正则非常常用，不过 search/match/reg.exec 连着来就有点忘记谁是谁了（尤其是加上 group 的匹配替换）

String

Search

返回子串起始位置，未找到则返回-1

let reg = /cola/g;
var str = "Drink cola";
var n = str.search(reg);

输出 6

Match

检索指定的值。

let reg = /c\S+a/g;
var str = "Drink cola";
var n = str.match(reg);

输出:["cola"]

let reg = /c\S+a/g;
var str = "Drink cola and cola";
var n = str.match(reg);
console.log(n);

输出 ["cola", "cola"]

Replace

replace 默认只替换第一个匹配，如果使用带全局的正则则全匹配：

let reg = /c\S+a/g; //带全局符号
var str = "Drink cola and cola";
var n = str.replace(reg, "pepsi");
console.log(n);

输出 Drink pepsi and pepsi，两个都被替换了。

另有replaceAll默认全匹配，如果使用不带全局的正则则会报错。

子句匹配：

let reg = /c(\S+)a/g;
var str = "Drink cola and cola";
var n = str.replaceAll(reg, "【$1】");
console.log(n);

输出 Drink 【ol】 and 【ol】

RegExp

RegExp 对象

var pattern = /cola/i;

Exec

let reg = /cola/g;
var str = "Drink cola";
var n = reg.exec(str);

输出 ["cola"]

命名子串:

let reg = /(?<year>\d{4})-(?<month>\d{2})/g;
var str = "2021-07";
var n = reg.exec(str);

打印n为：["2021-07", "2021", "07"]

n 的 object 为:

Array(3) [ "2021-07", "2021", "07" ]​
  0: "2021-07"​
  1: "2021"​
  2: "07"​
  groups: Object { year: "2021", month: "07" }​
  index: 0​
  input: "2021-07"​
  length: 3​
<prototype>: Array []

常见使用场景：循环匹配提取。

const str = "1.apple   2.banana    3.orange";
const reg = /(?<number>\d)\.(?<text>\S+)/g; // 像这样在while里使用的话，不要掉了全局标志！（否则无限循环
while ((m = reg.exec(str))) {
  console.log(m.groups.text);
}
