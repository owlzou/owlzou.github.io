Vue3 组件库组织学习

2022-03-05
2022-04-17
前端 vue JavaScript

出于学习的目的探究一下 Vue3 组件库的编写，刚好整个新 web app 用得上。

写组件本身问题不大，难受的是写文档（）目前主要探索一下文件组织。

建立项目

使用 vue3 + vite + typescript

新建项目

npm init vite-app round-ui

测试

  • vue-jest
  • vue-test-utils
yarn add -D jest ts-jest typescript @types/jest @types/jest @vue/vue3-jest @vue/test-utils@next @babel/preset-env @testing-library/jest-dom

@testing-library/jest-dom 用来添加一些方便 DOM 测试的函数。

测试设置初始化

yarn ts-jest config:init

修改生成的 jest.config.js ：

/** @type {import('ts-jest/dist/types').InitialOptionsTsJest} */
module.exports = {
  preset: "ts-jest",
  testEnvironment: "node",
  transform: {
    "^.+\\.js$": "babel-jest",
    "^.+\\.vue$": "@vue/vue3-jest", // Update to match your installed version
  },
};

修改 tsconfig.json:

{
  "compilerOptions": {
    "esModuleInterop": true
  }
}

在编写的测试文件顶部加入环境设置：

/**
 * @jest-environment jsdom
 */

文档

  • storybook，但目前 storybook 对 Vue(+ts) 的支持还不够完美。
cd round-ui
npx sb init
  • vite-press
yarn add --dev vitepress
  • vue-styleguidist 从 Vue 组件&注释中生成文档，也能利用 vue-docgen-cli 抽取的数据用在其他文档生成器中（比如 vuepress）。

vue-docgen-cli + vitepress 示例：

# 新建 docs 文件夹，在文件夹里
yarn add -D vue-docgen-cli vitepress

docs里新建 docgen.config.js, 按 设置 填写。

const path = require("path");

module.exports = {
  componentsRoot: "../packages/components", // the folder where CLI will start searching for components.
  components: "**/[A-z]*.vue", // the glob to define what files should be documented as components (relative to componentRoot)
  outDir: "docs", // folder to save components docs in (relative to the current working directry)
  getDestFile: (file, config) =>
    path.join(config.outDir, file).replace(/\.vue$/, ".md"), // specify the name of the output md file
};

运行抽取：

yarn vue-docgen

生成的 md 文档就会放到 docs/docs 下了。
